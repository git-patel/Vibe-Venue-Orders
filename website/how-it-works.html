Operational Guide & Architecture

A deep dive into exactly how our Flutter application powers offline ordering. Learn how vendors set up the environment and how attendees experience frictionless ordering.

The Vendor Command Center

  1. 1

    System Initialization & Profile Setup

    The vendor installs the Vibe Venue Android APK. Upon launching, they create a local profile, configure their Stall Name (e.g., "Main Stage Grill"), and optionally set up UPI Payment IDs and Text-To-Speech (TTS) preferences. All settings are saved to a local SQLite database.

  3. 2

    Menu Database Configuration

    The vendor navigates to "Manage Menu". Here they construct their inventory: adding items, assigning prices, and uploading images from local storage. Most importantly, they can create Custom Fields. For example, creating a "Condiments" checkbox array or a "Size" dropdown. This data is serialized and stored in SQLite.

  5. 3

    Network Bootstrapping (Hotspot & Shelf Server)

    The vendor turns on their Android Wi-Fi hotspot. In the app, they hit "Ordering Mode" to start the server. The app spins up a Dart shelf_io HTTP server on port 8080. A massive QR code is generated mapping to the server's local IP (e.g., http://192.168.43.1:8080/menu).

  7. 4

    Real-Time Order Handling & Rush Automation

    As orders roll in, the app triggers local device notifications and speaks the order details out loud using the TTS engine. The vendor uses the dashboard to tap order cards, moving them from Pending to Preparing to Ready.

    Pro Tip: If the line gets too crazy, the vendor enables Rush Mode, which automatically progresses orders through the pipeline based on configured minute timers to keep the queue flowing without manual screen tapping.

  9. 5

    Delivery & Payment Logging

    When the food is ready, a runner grabs it. The vendor taps "Deliver" and a popup appears to record exactly how it was paid (Cash or Online) and if any extra tip was provided. This writes a final timestamp to the DB, clearing it from the active dashboard and moving it to Analytics.

The Frictionless Attendee Journey

  1. 1

    Scan & Local Connect

    The attendee sees the printed QR code pasted to the back of their stadium seat or at a standing table. They scan it with their camera. The camera prompts them to connect to the "VibeVenue" Wi-Fi hotspot. Zero cellular data is consumed.

  3. 2

    Browser Rendering of Dynamic HTML

    Once connected, their browser automatically opens the IP address. The vendor's app intercepts the GET request and dynamically generates an optimized HTML payload built directly from the SQLite database. It loads instantly because the server is less than 50 feet away.

  5. 3

    Cart Customization

    The attendee adds items to their local JavaScript session cart. If they select a "Venue Burger", a modal pops up presenting the Custom Fields (like "No Onions", "Extra Sauce") defined previously by the vendor.

  7. 4

    Checkout & Post (JSON Payload)

    At checkout, the attendee types their Name, Phone, and Zone / Seat Number. When they hit submit, the browser fires a POST request with a JSON payload of their order to the vendor's /submit_order endpoint.

  9. 5

    Polling for Status

    The browser immediately redirects to a tracking page. This page contains a lightweight Javascript polling script that pings the vendor's /track/{order_id}/status endpoint every 30 seconds. The attendee sees their status change from Preparing to Ready, and simply waits for the runner to arrive with their food!
